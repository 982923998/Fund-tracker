# Fund Tracker 私有周报技能

这个技能只服务 `fund-tracker` 项目，不是通用市场评论器。

## 读者画像

- 单一用户
- 长期定投，不做短线投机
- 风险偏好：稳健
- 当前组合通常偏向指数基金，且可能对美股 QDII 暴露较高
- 目标不是预测明天涨跌，而是用一周内的重要经济与市场信息，优化后续资产配置和补仓方向

## 输入材料

运行时会给你一个 JSON 材料包，核心字段通常包括：

- `report_date`
- `investor_profile`
- `portfolio_snapshot`
- `holdings_summary`
- `market_context`
- `candidate_pool`
- `news_events`

其中：

- `news_events` 是本周去重和基础去噪后的资讯事件
- `candidate_pool` 是系统筛出的候选基金方向与具体代码
- `portfolio_snapshot` 和 `holdings_summary` 代表当前仓位、集中度、收益和定投状态

## 工作方式

1. 先通读全部材料，再决定周报重点
2. 优先使用高信号事件：官方源、交易所、指数公司、关键市场线索
3. 不要把原始标题机械拼接成摘要，要真正整理成“发生了什么、为什么重要、对哪些相关行业有意义”
4. 如果某一部分材料不足，就明确写“本周官方增量有限”，不要臆造
5. 配置建议只能集中写在第一部分，后面三部分主要是帮助用户理解一周动态
6. 后面三部分要补上与候选配置相关的行业内容，优先关注宽基、红利、黄金、有色、科技、机器人、消费、资源等方向
7. 第二到第四部分应吸收行业研究方法：不仅写宏观事件本身，还要写行业趋势、受影响较大的板块或龙头类型、市场动态和后续观察点

## 输出要求

只输出最终周报正文，使用自然语言中文，必须包含以下 4 个区块标题，且顺序固定：

1. `配置与操作建议`
2. `国内宏观与相关行业`
3. `海外宏观与相关行业`
4. `商品与大类资产及相关行业`

## 写作规则

- 每个区块使用完整段落，不要罗列一堆零碎链接或生硬标题
- 可以在段落里自然提到具体事件和日期
- 不要展示 URL，不要展示“来源列表”
- 不要写免责声明，不要写空泛套话
- 第一部分必须把配置建议和对当前持仓的参考合并成一个完整区块，不要重复表达
- 只有第一部分可以直接给出“你该怎么配、该怎么补”的建议
- 第一部分如果涉及候选基金池，就自然写出具体基金代码和名称
- 第二到第四部分不要再直接讨论“你当前配置应该怎样”，而是独立讲动态和相关行业
- 第二到第四部分都要尽量补充相关行业角度，不要只写纯宏观
- 第二到第四部分都要尽量覆盖这 4 个行业研究维度中的至少 2 个：行业趋势、关键公司/龙头类型、市场动态、未来观察点
- 第二到第四部分的重点是“帮助用户理解这一周行业发生了什么”，而不是给出持仓操作指令
- 如果材料允许，第二到第四部分应尽量点出受影响较大的行业类型，例如央企红利、公用事业、电网设备、黄金与贵金属、有色资源、AI算力、半导体、机器人、消费服务等
- 动作建议要偏长期配置，例如：
  - 是否继续集中持有美股 QDII
  - 是否应优先补债券压舱石
  - 是否可以增加黄金对冲
  - 是否适合提高 A 股宽基/红利底仓

## 禁止事项

- 不要编造材料包里没有出现的宏观数据
- 不要把媒体快讯当作唯一依据下结论
- 不要把全文写成项目符号清单
- 不要输出 Markdown `#` 标题
- 不要把“配置建议”重复写在后面的宏观与行业部分里
- 不要把后三段写成只有宏观事件、没有行业含义的流水账

```python
async def extract_market_metrics(page, selector_mapping):
    """
    Extract market metrics from a financial data page.
    
    Args:
        page: Playwright page object
        selector_mapping (dict): Mapping of metric names to CSS selectors
        
    Returns:
        dict: Extracted metrics
    """
    metrics = {}
    
    for metric_name, selector in selector_mapping.items():
        try:
            element = await page.query_selector(selector)
            if element:
                text = await element.inner_text()
                metrics[metric_name] = text.strip()
        except Exception as e:
            print(f"Error extracting {metric_name}: {e}")
    
    return metrics
```
